#  Reflection

This project pushed me beyond “making it work” toward building something reliable and polished. On the API side, I learned to design requests intentionally: scoping fields to meet RestCountries limits, reconciling LLM suggestions with official docs, and modeling optionals so the UI stays resilient when values are missing. Debugging improved when I adopted an incremental approach—logging raw responses, validating minimal decoders first, then layering fields—which ultimately fixed the empty list issue. On the client side, separating concerns with lightweight view models made async flows clearer, and using async/await with main-actor updates helped me surface loading and error states predictably. I tuned search and filtering for responsiveness without premature optimization. Visually, I iterated on a cohesive system: consistent corner radii, spacing, and subtle strokes; a dark palette with accent highlights; and Nunito typography that reads well at multiple sizes. Small touches—region badge colors, hero header, and card borders—made the app feel intentional. I also considered accessibility by checking contrast in dark mode and keeping touch targets generous, and I kept scrolling smooth with lazy stacks and lightweight images. A valuable lesson was learning how to collaborate with AI: it’s great for scaffolding and exploring alternatives, but I now treat its output as a starting point, verifying against documentation, adding targeted logs, and building features in small, testable increments. I’m proud that attractions data appears correctly and that the experience feels coherent from the searchable list to the detail view. With more time, I’d expand persistence (favorites, travel notes), add richer map interactions, and write a small set of tests around networking and decoding. Overall, this work strengthened my skills in API integration, state-driven UI, design polish, and systematic debugging—and gave me a repeatable approach I can apply to future apps.
